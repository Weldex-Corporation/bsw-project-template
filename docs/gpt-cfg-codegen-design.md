# Gpt MCAL Cfg codegen — design sketch

Status: design only, not implemented. Follow-up after bsw-mcal-msp PR #5
and bsw PR #209 merge.

Today (after PR #38) the rte_os tick channel is wired by a hand-written
`Gpt_ConfigType` literal inside `EcuM_bcc1.c`:

```c
static const Gpt_ChannelConfigType ecum_gpt_channels[] = {
    { .channel      = 0u,
      .mode         = GPT_CH_MODE_CONTINUOUS,
      .notification = EcuM_Bcc1_TickNotification },
};
static const Gpt_ConfigType ecum_gpt_config = { … };
```

That's fine for the template but the same place every other MCAL config
(Port, Dio, Uart, Pwm, Adc, Mcu …) is generated from the board YAML by
`bsw-mcal-msp/tools/gen_cfg.py`. Bringing Gpt under the same flow keeps
multi-board templates consistent and stops EcuM from hard-coding a
specific channel ↔ timer mapping.

## YAML schema (proposed)

```yaml
gpt:
  # Predef Timer 1 µs / 32-bit (SWS_GPT_00373).
  # Free-running, drives Gpt_GetPredefTimerValue.
  predef:
    timer:        TIMG8

  # Channel notifications (Gpt_EnableNotification).
  channels:
    - name:         SystemTick
      timer:        TIMG7
      mode:         CONTINUOUS         # or ONESHOT
      period_us:    1000               # → 1 ms
      notification: EcuM_Bcc1_TickNotification
```

## Generated `Gpt_PBcfg.c` (sketch)

```c
/* Generated by tools/gen_cfg.py from boards/lp_mspm0g3507.yaml.
 * Do not edit by hand. */
#include "Gpt.h"

extern void EcuM_Bcc1_TickNotification(void);

#define GPT_CH_SYSTEM_TICK   ((Gpt_ChannelType)0u)

static const Gpt_ChannelConfigType Gpt_PBcfg_channels[] = {
    {
        .channel      = GPT_CH_SYSTEM_TICK,
        .mode         = GPT_CH_MODE_CONTINUOUS,
        .notification = EcuM_Bcc1_TickNotification,
    },
};

const Gpt_ConfigType Gpt_PBcfg = {
    .channels    = Gpt_PBcfg_channels,
    .numChannels = (uint8)(sizeof(Gpt_PBcfg_channels) / sizeof(Gpt_PBcfg_channels[0])),
};
```

## Generated `Gpt_Cfg.h` (sketch)

```c
#ifndef GPT_CFG_H
#define GPT_CFG_H

#include "Std_Types.h"

/* Channel logical IDs */
#define GPT_CH_SYSTEM_TICK   ((Gpt_ChannelType)0u)

/* Predef Timer backing (informational — vendor's Gpt_Hw.c may use this) */
#define GPT_PREDEF_TIMER_HW  TIMG8

/* Tick period reference (used by EcuM_Bcc1 when calling Gpt_StartTimer) */
#define GPT_SYSTEM_TICK_PERIOD_US  1000u

extern const Gpt_ConfigType Gpt_PBcfg;

#endif
```

## EcuM_bcc1.c after migration

```c
#include "Gpt_Cfg.h"

void EcuM_Init(void) {
    Mcu_Init();
    Gpt_Init(&Gpt_PBcfg);              /* from generated config */
    debug_init();
    Det_Init();
    Gpt_StartTimer       (GPT_CH_SYSTEM_TICK, GPT_SYSTEM_TICK_PERIOD_US);
    Gpt_EnableNotification(GPT_CH_SYSTEM_TICK);
}

/* The inline ecum_gpt_channels[] / ecum_gpt_config and the
 * ECUM_BCC1_TICK_* defines drop out. */
```

## `gen_cfg.py` addition (sketch)

Add a `gen_gpt()` function next to `gen_mcu()` and call it from `main()`:

```python
def gen_gpt(cfg, chip, board, out_dir):
    gpt = cfg.get('gpt') or {}
    channels = gpt.get('channels', [])
    predef   = gpt.get('predef', {})

    h_lines  = [_header(board, chip, 'Gpt'), '#include "Std_Types.h"\n\n']
    c_lines  = [_header(board, chip, 'Gpt'),
                '#include "Gpt.h"\n\n']

    for idx, ch in enumerate(channels):
        h_lines.append(f"#define GPT_CH_{ch['name'].upper()}   "
                       f"((Gpt_ChannelType){idx}u)\n")
        c_lines.append(f"extern void {ch['notification']}(void);\n")
        h_lines.append(f"#define GPT_{ch['name'].upper()}_PERIOD_US  "
                       f"{ch['period_us']}u\n")

    # ... emit Gpt_PBcfg_channels[] + Gpt_PBcfg ...
    _write(out_dir, 'Gpt_Cfg.h',   ''.join(h_lines))
    _write(out_dir, 'Gpt_PBcfg.c', ''.join(c_lines))
```

## Risks

- Channel-name → notification-function-name coupling is fragile if the
  notification lives in `EcuM_bcc1.c` (template) while the
  `Gpt_PBcfg.c` lives in the generated tree. Document the convention
  loudly in the YAML and the header.
- `Gpt_Hw.c` (vendor) currently hard-codes TIMG7 ↔ channel 0 and
  TIMG8 ↔ Predef. The YAML lets you express other mappings (e.g.
  TIMG14 for chips without TIMG7) but **only** if `Gpt_Hw.c` is also
  taught to look at `Gpt_Cfg.h` rather than its own `#define`s.
  This is a separate refactor in `bsw-mcal-msp/src/Gpt_Hw.c`.
